# State

## Current Focus

Work is on branch `feat/statusline-auto-install-go-hook` (built on the PR-#42 statusline panel work). Three vertical phases are COMPLETE and committed on this branch:

**(1) tty-safe renderer** — `crafter statusline` run manually in a terminal returns promptly instead of blocking on `io.ReadAll(os.Stdin)`; under Claude Code's pipe the output is byte-identical.

**(2) Node-free `crafter check-update` hook** — the Node SessionStart hook was ported 1:1 to a Go subcommand (`cli/cmd/check_update.go`): silent-fail, exit 0, non-blocking detached background refresh, byte-compatible cache contract (same cache path/fields, 4h/14400s freshness window, notice text, GitHub `releases/latest` endpoint, `v`-strip + strict semver as the deleted JS). `install.sh install_hook()` now registers `"<crafter_bin>" check-update` instead of `node …`; `hooks/crafter-check-update.js` is deleted; the hook no longer depends on `node` being on PATH (also fixes the bare-`node`-not-on-PATH case). `claudesettings.Save()` gained `MkdirAll` to stay self-sufficient after the installer stopped creating `~/.claude/hooks`.

**(3) Statusline installs by default** — both `--global` and `--local` runs run the existing smart-replace decision tree on every install (absent→set, ours→update/no-op, foreign→ask on a real TTY with `.bak` + echoed old command / keep + ready-to-paste guidance when non-interactive, never hang); the `--with-statusline` opt-in flag was removed entirely (now an unknown flag that hard-errors). Install tests inverted to default-on (real-binary harness) + a new flag-rejection test; README + `.crafter/ARCHITECTURE.md` describe the new default in neutral, behavior-focused wording.

PR for this branch is open as **PR #44** (https://github.com/richardriman/crafter/pull/44). Separately, the progressive-loading task remains parked on branch `feat/crafter-do-progressive-loading` (Phase 0+1 done; Phase 2 — orchestrator master-plan + lazy per-step `rules/do/*` loading — still pending; see `## Planned`).
